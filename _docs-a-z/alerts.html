Alerts help draw attention to brief confirmation or warning messages—or any other important information—that shouldn't be overlooked while your visitor scans the page.

When modifying alert colors, be sure to check that your colors meet a sufficient contrast ratio. The colors provided in the Framework already do.

Never rely on color alone to convey meaning. Include a visual icon in alerts for those who experience colors differently.

Confirm and Warn

The NYC Core framework provides two types of alerts: Confirm implied with green, and Warn implied with red.

Every instance of an alert should include an icon to help convey meaning. Across the Framework NYC Core Icons, our open-source icon library, is used for all icons.

Inverse Color Modifier

For a bolder look, chain the --inverse to alert--confirm or alert--warn modifier classes respectively.

Dismissable Alerts

For dismissable alerts, add the .alert--dismissable modifier class. Doing so places a close button inside the .alert.

When the close button is pressed, the alert fades out, then the entire .alert selector is removed from the DOM.

More Style Variations

Looking for even more alert styles? Exclude the .alert--confirm or .alert--warn modifier classes. Instead, add the color utilities or border utilities for different accents and style variations.
