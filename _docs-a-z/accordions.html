Every so often the the need for accordions will arise. Hovever, accordions should be used judiciously and under special circumstances. Accordions are most practical when only specific content might be relevant to a visitor—like an FAQ page, or when several topics are covered on a single page. It's best to avoid using accordions to simply make long pages appear shorter.

The Accordion Selector

In the NYC Core Framework, accordions are treated as a group within the .accordion selector. This class is required for the accordions to properly close one another.

Each subsequent accordion inside of .accordion includes a selector called .accordion__button. This is the accordion's header and triggers the accordion panel to open and close.

The next selector, .accordion__panel, holds the content associated with its preceding accordion header.

Be aware that custom padding directly applied to the .accordion__panel selector will not work. If you wish to apply custom padding to the accordion panel, either with CSS or one of our predefined padding classes, you may do so on the .accordion__panel selector's child <div>, which includes the data-accordion="content" attribute (see Figure 1, line 8).

Accordion Headers

Semantically, accordion headers should match the information architecture of the page. For example, if an accordion group lives directly below a page's <h1> header tag, the accordion headers should be <h2> tags, or have an aria-level="2" attribute present.
