Primary Nav Component

Designed for use within the document's primary header tag, our primary nav component is a complete menu system made up of three preassigned areas: the logo (wrapped within a home link), global action buttons, and of course, the menu itself.

The primary nav's parent selector requires two chained modifiers (.primary-nav--{x}--{y}) where x provides the menu placement (as it relates to the logo inline or below), and y provides the responsive breakpoint (e.g., lg) at which the component changes layout from mobile to desktop. With the example above, the parent selector class is .primary-nav--below--lg.

The primary nav component requires three immediate child selectors: .primary-nav__logo contains the website's logo and home link, .primary-nav__menu contains the website's main menu, and .primary-nav__buttons, contains the website's global action buttons (such as language, search, or login).

List Item Spacing

With the .nav component, list items are separated with margin using relative (em) units. This way, the spacing of list items adjusts to the list's font-size, whether inherited or explicit. In some cases, such as a list of smaller buttons, you may want more concise spacing (using rem units) between list items.

On the .nav selector, add the modifier class .nav--gap-{x}, where x equals one of the available spacing units (similar to the grid-gap modifiers).

With the example below, the first list is without a gap modifier and has default spacing, vis-a-vis the second list example where the .nav--gap-1 modifier is present, creating tighter, non-relative space between the list items.
